## 二、9 项技术决策确认

### 决策 1：后端框架 → FastAPI ✅

| 维度 | 选择 | 原因 |
|------|------|------|
| 框架 | **FastAPI** | 原生 async/await，RAG 多异步 IO 刚需 |
| 替代方案 | Flask (淘汰) | 同步模式，需额外 Quart 扩展 |
| 文档 | Swagger UI 自动生成 | 开发调试零成本 |
| 校验 | Pydantic 自动 | 请求体/响应体类型安全 |
| 部署 | Uvicorn | ASGI 高性能服务器 |

**当前状态**：已部署，端口 8001，SSE 流式响应正常。

---

### 决策 2：向量数据库 → ChromaDB ✅

| 维度 | 选择 | 原因 |
|------|------|------|
| 数据库 | **ChromaDB** | `pip install` 一行部署 |
| 替代方案 | Milvus / Pinecone | 太重/付费 |
| 量级 | 几百~几千文档 | 绰绰有余 |
| 持久化 | PersistentClient | 重启不丢数据 |
| 相似度 | **cosine**（余弦距离） | 适配 instructor-xl 高基线特性 |
| 可重建 | 删 `chroma_db/` 目录即重置 | 运维友好 |

**关键参数**：
- `hnsw:space = cosine`（非默认 L2）
- `RAG_THRESHOLD = 0.69`（instructor-xl 余弦基线 0.64-0.66）

---

### 决策 3：嵌入模型 → instructor-xl（本地部署）✅

| 维度 | 选择 | 原因 |
|------|------|------|
| 模型 | **instructor-xl** | 本地部署，零外部依赖 |
| 替代方案 | text-embedding-3-small | 需 API Key + 联网 |
| 框架 | sentence-transformers 5.6.0 | 与 ChromaDB 深度集成 |
| 向量维度 | 768 | |
| 中文效果 | 良好 | 实测余弦相似度基线 0.64-0.66 |
| 共享复用 | `embedding.py` 单例 | LangChain + ChromaDB 共用 |

**部署**：MacBook Pro M1 本地，首次加载 ~3s，后续毫秒级。

---

### 决策 4：LLM 推理 → DeepSeek Cloud API ✅

| 维度 | 选择 | 原因 |
|------|------|------|
| 模型 | **deepseek-chat** (671B MoE) | 回答质量接近 GPT-4 |
| 调用方式 | langchain-openai (ChatOpenAI) | 兼容 OpenAI API 格式 |
| 成本 | ￥1/百万 token | 学习阶段几乎免费 |
| 异步 | `asyncio.to_thread` | 不阻塞 FastAPI 事件循环 |
| 替代方案 | Qwen2.5:3B (本地) | 回答质量中等，已弃用 |

**注意**：远程 vLLM (Qwen3-8B-sft) 作为备用推理后端，端口 20994。

---

### 决策 5：文件解析方案 ✅

| 文件类型 | 解析工具 | 方法 |
|----------|----------|------|
| PDF | **PyMuPDF** (fitz) | `page.get_text()` 逐页提取 |
| DOCX | **python-docx** | `doc.paragraphs` 段落提取 |
| TXT | 内置 `open()` | 直接读取 |
| HTML | **BeautifulSoup** | `get_text()` 去标签 |
| PPTX | **python-pptx** | `slide.shapes` 文本框提取 |
| IPYNB | **json** (标准库) | 解析 cells (code/markdown) |

**上传限制**：50MB，自动分块入库 ChromaDB。

---

### 决策 6：RAG 框架 → LangChain ✅

| 维度 | 选择 | 原因 |
|------|------|------|
| 框架 | **LangChain** | `RetrievalQA` 一行搭链路 |
| 检索 | `similarity_search_with_relevance_scores` | Top-3 + 分数过滤 |
| 分块 | `RecursiveCharacterTextSplitter` | chunk=500, overlap=50 |
| Prompt | 自定义 SYSTEM_PROMPT | 闲聊兜底 + 课程专区 |
| 编排 | LangChain Expression Language | 灵活可扩展 |

**分块配置**：
```
chunk_size = 500
chunk_overlap = 50
separators = ["\n\n", "\n", "。", "！", "；", "，", " "]
```

---

### 决策 7：前端框架 → Gradio 6.18 ✅

| 维度 | 选择 | 原因 |
|------|------|------|
| 框架 | **Gradio 6.18** | Chatbot 原生聊天组件 |
| 替代方案 | Streamlit | 聊天需第三方库 |
| 流式 | `httpx.stream()` + yield | 打字机效果 |
| 多会话 | gr.State + 字典管理 | 按日期分组 |
| CSS | 玻璃拟态顶栏 + 渐变色侧边栏 | 面试展示效果好 |
| 文件上传 | gr.File | 拖拽上传 |

**界面组件**：
- 左侧栏：新建/切换/删除会话，日期分组
- 顶栏：欢迎语 + 退出按钮 + 项目标题
- 中区：Chatbot 流式显示 + 路由标记

---

### 决策 8：业务数据库 → SQLite ✅

| 维度 | 选择 | 原因 |
|------|------|------|
| 数据库 | **SQLite** | Python 标准库内置 |
| 部署 | 单文件零配置 | 本地完美契合 |
| 替代方案 | MySQL/PostgreSQL | 过度设计 |
| 存储内容 | 会话历史（最近 10 轮）+ 用户名 | |
| 键 | session_id (UUID) | 前后端同步 |

---

### 决策 9：意图路由 → 嵌入向量余弦相似度 ✅

| 维度 | 选择 | 原因 |
|------|------|------|
| 方案 | **余弦相似度 + 会话级缓存** | 一次嵌入 = 路由 + 检索 |
| 替代方案 | LLM Function Calling | 延迟 1-3s，消耗 token |
| 锚点 | 课程锚点 + 闲聊锚点 | instructor-xl 预计算 |
| 阈值 | `RAG_THRESHOLD = 0.69` | 精准过滤无关检索 |
| 缓存 | 同会话仅首次计算嵌入 | 避免 7-15s 重复计算 |

**路由逻辑**：
```
if max_score >= RAG_THRESHOLD and route == "course":
    → RAG 检索 + 生成，标记 📚 参考
else:
    → 直接 LLM 对话，标记 💬 闲聊
```
