## 三、模块一：RAG问答系统（第二周）

### 3.1 混合检索架构

```
用户查询 "什么是LoRA微调？"
        │
        ├──→ 稠密检索（ChromaDB cosine）   → Top-10
        ├──→ 稀疏检索（BM25 + jieba分词）  → Top-10
        │
        └──→ RRF 融合（k=60）             → Top-5
                 │
                 └──→ 可选：Cross-Encoder 重排序 → Top-3
```

### 3.2 关键设计

#### HybridRetriever（`src/rag/retriever.py`）

```python
class HybridRetriever:
    """稠密 + 稀疏 + RRF 融合"""
    def __init__(self, vectorstore, embedding_fn): ...
    def build_bm25_index(self): ...       # 从 ChromaDB 全量构建 BM25
    def rebuild_bm25_index(self): ...     # 文件上传后增量重建
    def retrieve(self, query, top_k=5):   # 主检索接口
        # 1. 稠密检索（ChromaDB）
        # 2. 稀疏检索（BM25）
        # 3. RRF 融合返回 Top-k
```

**RRF 公式**：`RRF(d) = Σ 1/(k + rank_i(d))`，k=60

#### BM25Index — 中文分词

- 使用 `jieba` 精确模式分词
- 启动时从 ChromaDB 全量文档构建
- 文件上传后增量重建

### 3.3 提示词模板化（`src/rag/prompts.py`）

- 结构化引用格式：回答中使用 `[1]` `[2]` 标注来源
- 回答末尾「📚 参考来源」列表
- 使用 Python `string.Template` 管理模板

### 3.4 查询扩展与缓存

| 组件 | 功能 |
|------|------|
| `QueryCache` | LRU 内存缓存（100条），MD5 精确匹配 |
| `QueryExpander` | LLM 生成 3 个查询变体，多路召回取并集 |
| `QueryCondenser` | 多轮对话中，压缩上下文为独立检索查询 |
| `RetrievalEvaluator` | Recall@K, Precision@K, MRR 评估 |


## 四、模块二：文件解析处理（第二周）

### 4.1 统一接口

```python
@dataclass
class ParseResult:
    text: str                          # 完整文本
    tables: List[str]                  # 表格（Markdown 格式）
    metadata: dict                     # 标题/作者/页码/章节
    chunks: List[dict]                 # 预分块（含元数据）
    errors: List[str]                  # 非致命错误

class BaseParser(ABC):
    def parse(self, file_path: str) -> ParseResult: ...
    def supported_extensions(self) -> List[str]: ...
```

### 4.2 表格提取

| 格式 | 方法 |
|------|------|
| PDF | `page.find_tables()` → Markdown 表格 |
| DOCX | `doc.tables` → Markdown 表格 |
| CSV | `csv.reader` → 自动识别表头 |

失败时降级为纯文本（非致命错误）。

### 4.3 按文档类型智能分块

| 文档类型 | 分块策略 |
|---------|---------|
| PDF | 先按页分割 → 页内再按 chunk_size 细分 |
| DOCX | 保留标题样式层级（Heading 1/2/3） |
| Markdown | 按 `# / ## / ###` 标题分割 |
| CSV | 概览描述 + 表格数据 chunk |
| 通用 | RecursiveCharacterTextSplitter（500/50） |

### 4.4 元数据保留

每个 chunk 携带：`source`、`chunk_index`、`heading_path`（如 "三、vLLM > 3.2 PagedAttention"）、`page_number`（PDF）、`document_title`

### 4.5 支持的格式

| 格式 | 扩展名 | 解析器 |
|------|--------|--------|
| PDF | .pdf | `PDFParser` |
| Word | .docx | `DocxParser` |
| PowerPoint | .pptx | `PPTXParser` |
| HTML | .html | `HTMLParser` |
| Jupyter | .ipynb | `IPYNBParser` |
| 纯文本 | .txt | `TXTParser` |
| Markdown | .md | `MarkdownParser` ✨新增 |
| CSV | .csv | `CSVParser` ✨新增 |

### 4.6 新增依赖


## 五、模块三：意图识别系统（第三周）

### 5.1 五类意图

```python
class Intent(str, Enum):
    COURSE_QUESTION = "course_question"   # AIGC 技术问题 → RAG
    CASUAL_CHAT = "casual_chat"           # 闲聊/问候
    FILE_OPERATION = "file_operation"     # 文件上传/解析
    WEB_SEARCH = "web_search"            # 需要联网搜索
    SYSTEM_COMMAND = "system_command"     # 系统操作/帮助
```

### 5.2 混合分类策略

不采用纯嵌入质心分类（instructor-xl 对细粒度意图区分度不足，各类质心余弦相似度 > 0.82），改用混合策略：

```
Step 1: ChromaDB 检索分数 → 判断是否课程相关问题
Step 2: 关键词/正则匹配 → 细分非课程意图
         - Web Search: 匹配 "搜索/查一下/天气/新闻/最新..."
         - File Operation: 匹配 "上传/解析/分析...文件/文档/PDF..."
         - System Command: 匹配 "列出/新建...会话/帮助..."
Step 3: 技术术语检测 → LoRA/vLLM/Transformer/RAG...
Step 4: LLM 二次验证 → 中置信度时确认
Step 5: 回退链 → 低置信度时逐层回退
```

### 5.3 决策优先级

```
1. 明确搜索关键词     → web_search   (conf=0.8)
2. 明确文件操作关键词 → file_operation (conf=0.8)
3. 明确系统命令关键词 → system_command (conf=0.8)
4. ChromaDB 高分匹配  → course_question (conf=rag_score)
5. 检测技术术语       → course_question (conf=0.55)
6. 默认              → casual_chat    (conf=0.5)
```

### 5.4 回退链

```
高置信度 (≥0.55) → 直接执行
中置信度 (0.35-0.55) → LLM 二次验证
低置信度 (<0.35) → 回退链:
    1. 检测技术术语 → COURSE_QUESTION
    2. 默认 → CASUAL_CHAT
```

### 5.5 意图专属 System Prompt

每个意图类有独立的 System Prompt 模板，确保不同场景下回答风格适配。

## 六、模块四：网络搜索模块（第三周）

### 6.1 搜索引擎

```
WebSearchEngine
  ├── 主引擎：DuckDuckGo（免费，无需 API Key）
  ├── 备用引擎：Bing Web Search API（可选，需 API Key）
  └── 回退策略：主 → 备用 → 空结果
```

### 6.2 缓存与限流

| 组件 | 设计 |
|------|------|
| `SearchCache` | TTL 缓存（默认 1 小时），按 MD5 键存储 |
| `RateLimiter` | 令牌桶限流（默认 30 次/分钟），超限等待或拒绝 |

### 6.3 RAG 集成

```
用户问题 → 意图路由
  ├── WEB_SEARCH → 纯网络搜索
  ├── COURSE_QUESTION + RAG 分数低 → RAG + 网络搜索（合并上下文）
  └── 其他 → 按原逻辑
```

搜索结果在回答中标注 `[网页] 标题: URL`，与知识库来源 `[知识库]` 区分。

### 6.4 新增 API

```
POST /search  {query, max_results}
  → {query, source: "cache"|"live", results: [...], count: N}
```

### 6.5 新增依赖

```
duckduckgo-search>=7.0  # 免费网络搜索
```
