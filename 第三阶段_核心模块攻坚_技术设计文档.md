## 二、架构设计

### 2.1 模块目录

```
src/

│   ├── models.py      #   UserRegister/UserLogin/TokenResponse
│   ├── service.py     #   bcrypt + JWT 编解码
│   └── dependencies.py #  get_current_user 依赖注入
├── intent/            # 意图识别
│   ├── classifier.py  #   LLM分类 + k-NN回退 + 快速规则
│   ├── fallback.py    #   回退链
│   ├── router.py      #   IntentRouter 统一入口
│   ├── labeler.py     #   [新增] LLM标注数据生成器
│   └── prompts.py     #   意图专属 System Prompt
├── rag/               # RAG问答
│   ├── retriever.py   #   HybridRetriever + BM25Index

│   ├── cache.py       #   QueryCache LRU 100条
│   ├── prompts.py     #   结构化引用模板 [1][2]
│   ├── query_expander.py # QueryExpander/Condenser
│   └── evaluator.py   #   RetrievalEvaluator

└── session/           # 会话管理（SQLite + user_id隔离）
```

### 2.2 网络搜索架构（最终方案）

```
WebSearchEngine
  ├─ Bocha AI Search（主）  ← DeepSeek官方搜索供应商
  │   ├─ search()          ← 普通搜索
  │   └─ search_stream()   ← 流式，边搜边推token
  ├─ Bing 中国版（回退）    ← 爬虫，免费
  └─ Bing API（兜底）      ← 付费
```

**搜索引擎选型历程**：
```
DuckDuckGo（被墙）→ Kimi API（不暴露搜索）→ 爬百度（反爬）→ 博查 ✅
```

### 2.3 意图分类架构（最终方案）

```
用户消息 → 快速规则(城市/赛事,0延迟)
              │ 未命中
              ▼
          LLM分类(DeepSeek,理解语义+对话历史)
              │ LLM失败
              ▼
          k-NN回退(嵌入+ChromaDB,500条样本)
```

**分类器选型历程**：
```
关键词正则(95行,反复调) → 嵌入k-NN(依赖样本) → LLM分类✅
```

### 2.4 性能优化

```
并行: asyncio.create_task(意图分类, 搜索请求)
流式: Bocha stream → 3s首token → 逐字推送
缓存: QueryCache LRU 100条
快速路径: 城市名/赛事追问规则拦截(0延迟)
```

## 四、混合检索架构（RAG）

```
用户查询 → ├─ 稠密检索 (ChromaDB cosine)  → Top-10
           └─ 稀疏检索 (BM25 + jieba分词)  → Top-10
                    ↓
              RRF 融合 (k=60)              → Top-5
                    ↓
              RAGGenerator (流式 + 结构化引用[1][2])
```

## 六、关键设计决策

| 决策 | 理由 |
|------|------|
| Bocha 而非 Bing 爬虫 | Bocha 是 DeepSeek 官方搜索供应商，中文质量高，有结构化数据 |
| Bocha 回答直接用，不经过 DeepSeek | 防止 LLM 二次总结时编造（幻觉） |
| LLM 做意图分类，不用 k-NN | LLM 理解语义 + 对话历史，准确率更高 |
| 并行化意图+搜索 | 不互相依赖，并行节省 2-3s |
| 流式输出 | 3s 首 token vs 9s 完整等待 |
| JWT 认证 | 会话隔离，保护用户数据 |
| bcrypt 替代 passlib | passlib 和 bcrypt 4.x 不兼容 |
